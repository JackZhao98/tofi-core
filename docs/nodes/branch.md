# Branch Node

> Route execution based on a boolean condition. Usually auto-generated by Compare/Check nodes.

**Type:** `branch`
**Category:** Logic (Internal)
**Source:** `internal/engine/logic/branch.go`

---

## Config

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `condition` | string | Yes | Value to evaluate. Typically `{{compare_node}}` or `{{check_node}}`. |
| `on_true` | string[] | No* | Node IDs to execute when condition is truthy |
| `on_false` | string[] | No* | Node IDs to execute when condition is falsy |

*At least one of `on_true` or `on_false` must be defined.

## Truthiness Rules

| Value | Result |
|-------|--------|
| `"true"` (case-insensitive) | truthy |
| `"1"` | truthy |
| `"yes"` (case-insensitive) | truthy |
| Any other non-empty string | truthy |
| `"false"` (case-insensitive) | falsy |
| `"0"` | falsy |
| `"no"` (case-insensitive) | falsy |
| `""` (empty string) | falsy |

## Output

Returns the condition value prefixed with routing info:
- `BRANCH_TRUE:true` — engine routes to `on_true` nodes
- `BRANCH_FALSE:false` — engine routes to `on_false` nodes

## Frontend Behavior

Branch nodes are **not directly created** in the UI. Instead:
1. User configures `on_true`/`on_false` on a Compare or Check node
2. The serializer auto-generates a `{node_id}_branch` node
3. The deserializer merges it back on load

## Example

```yaml
# Typically paired with compare/check:
score_check:
  type: compare
  config:
    left: "{{score}}"
    operator: ">"
    right: "80"
  next: ["score_check_branch"]

score_check_branch:
  type: branch
  config:
    condition: "{{score_check}}"
    on_true: ["high_score_handler"]
    on_false: ["low_score_handler"]
```
